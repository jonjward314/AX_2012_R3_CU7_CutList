1. Overview

SR423365 adds two new fields — Per_Series and Pieces — to the cutlist subsystem so that the SSRS cutsheet report can calculate and display per-series quantities for manufacturing orders.

New FieldSource BOM FieldTypePurpose
Per_Series NEW BOM.BOMQtySerie REAL Quantity produced per manufacturing series
Pieces NEW BOM.McaPieces INT Number of pieces per BOM line

The key calculated report expression added to the SSRS report:

Ceiling( Sum(Pieces × KitQty) / Per_Series )

Rounded up, with Per_Series = 0 handled as 1 (safe divide).

4.2 Sales-Order Path — McaCutlistCreate

doCutlistItems() builds an 8-table join query per sales line, then calls createCutlistRecord() for each BOM raw-material line found.

SalesLine (RecId, InventDimId, DueDate...)
    │
    └─ doCutlistItems() — 8-table join
            │
            ├─ ReqItemTable       (item coverage → site/location)
            ├─ InventLocation     (warehouse)
            ├─ InventDim [1]      (site, location, WMS)
            ├─ BOMVersion         (model BOM header)
            ├─ InventDim [2]      (BOM version dims)
            ├─ BOM raw-material line
            │     ├─ BOMQty       → Qty
            │     ├─ BOMQtySerie  → Per_Series  <<< SR423365
            │     ├─ McaPieces    → Pieces       <<< SR423365
            │     ├─ Height / Width / Depth
            │     ├─ McaMachine   → WrkCtrId
            │     └─ McaPattern   → Pattern
            ├─ InventDim [3]      (raw material dims)
            └─ InventTable        (McaUsageType, UnitId)

    Per BOM line → createCutlistRecord():
        ├─ DocuRef lookup     (notes)
        ├─ InventDim lookup   (site/location)
        ├─ CutGroupId         (McaCutlist::lookupCutGroup)
        ├─ MoveToAreaId       (McaCutlist::lookupMoveToArea)
        ├─ BluePrint          (McaEcoResProductPrints if PrintNumber set)
        └─ McaCutlist.insert()

Date filtering on BOM lines:

(BOM.fromDate <= DateOffLine  OR  !BOM.fromDate)
AND
(BOM.toDate   >= DateOffLine  OR  !BOM.toDate)

Fallback strategy: if the primary item-coverage join returns no rows, two fallback queries run in sequence (parent BOM site, then direct component site lookup).

4.4 Report Path — McaCutlistReportDP → McaCutlistBaseReport

McaCutlistReportDP.processReport() queries McaCutlist, populates McaCutlistTmp, then SSRS renders the report using two new SR423365 expressions:

ExpressionPurposeFormula
Qty per series Calculate how many series worth of cuts are needed =Ceiling( Sum(Pieces * KitQty) / IIf(Per_Series <= 0, 1, Per_Series) )
Per_Series display Show value or friendly fallback =IIf(Per_Series <= 0, "Not Set", Per_Series)

Report grouping: Model → Prefix → CutGroup, with page breaks on group changes. Detail rows show one line per McaCutlistTmp record.

8. Potential Gap — Module Path Per_Series

Verify during Step 6 DEV testing: McaCutlistCreate_Module.createCutlistRecord() sets Pieces but does not set Per_Series. The base class sets both.

Two possible explanations — confirm with Kelly Gordon / Rhonda Rowe:

  1. Intentional: all module items are one-per-series, so Per_Series defaults to 0 / "Not Set" is acceptable on the report.
  2. Omission: the assignment cutlist.Per_Series = bomRawMaterial.BOMQtySerie; was missed in the module subclass override and should be added.
