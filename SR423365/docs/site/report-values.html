Summary & Cover equations

Production: Total Units = Sum of KitQty
Dev: Total Units = Ceiling(Sum of KitQty)

Plain language: if total is 10.2, Production shows 10.2 while Dev shows 11.

Detail equation

Production legacy pattern: Total ≈ Sum(Pieces) × KitQty (or Sum(Pieces) in some regions)
Dev: Total = Ceiling( Sum(Pieces × KitQty) ÷ Per_Series )
Dev fallback: if Per_Series is missing or 0, divide by 1 and show “Not Set”.

Plain language: Dev introduces a per-series divisor, so totals can go down when Per_Series is > 1.

Why stakeholders see differences

  • DEV has extra fields (`Per_Series`, `PWSBuildScheduleId`, `UnitCounts`) available to expressions.
  • DEV rounds some totals up with Ceiling.
  • DEV detail math uses per-series normalization, changing quantity behavior.
  • Different report regions may still use different formulas.
