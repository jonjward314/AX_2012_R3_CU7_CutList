SR423365 Cutlist System — Production vs Current Dev

Comparison website focused on how data is generated, transformed, compacted/grouped, and rendered into the 3 report outputs.

Executive Summary Data Flow (Mermaid) Compaction/Grouping 3 Reports Field Lineage Key Differences

1) Executive Summary

Production behavior: `Pieces` + `KitQty` drive report totals. `Per_Series` is not active in the analyzed production SSRS data path.
Detail DP transform: `mcaCutlistTmp.Pieces = real2int(cutlist.Pieces * cutlist.KitQty)` happens before SSRS.
Current Dev behavior: adds `Per_Series` in salesline create path and SSRS formula `Ceiling(Sum(Pieces*KitQty)/Per_Series)` with safe fallback.
Current Dev gap: module create path still does not set `Per_Series`, so denominator fallback (1) can inflate outputs.

4) The 3 Reports — Individual Treatment

WorkOrderSummary

PROD: Uses summary DP rowset and legacy quantity logic (no active per-series denominator path).
DEV: Carries `Per_Series` and added display fields, but may still include legacy expression regions.

WorkOrderCoverSheet

PROD: Aggregated cover rowset with group keys such as model/prefix/site/move-to/date.
DEV: Same base path with additional Per_Series/unit-count context carried through.

WorkOrderDetail

PROD: DP pre-expands pieces (Pieces*KitQty), then SSRS sections use legacy totals.
DEV: Adds Ceiling(Sum(Pieces*KitQty)/safe(Per_Series)). If `Pieces` is already expanded and multiplied again in SSRS, effective numerator can over-scale.

5) Field Lineage Comparison

FieldProductionCurrent DevRisk

KitQty Set from explode/module qty, copied to temp and used in expressions. Same. Medium when mixed with expanded Pieces.

Per_Series Not active in analyzed production SSRS path. Salesline path sets it; module path still missing assignment. High: denominator fallback to 1 can inflate totals.

6) Key Differences Highlighted

  1. Schema/Expression: DEV report dataset and expressions include `Per_Series`; PROD does not.
  2. Creation asymmetry in DEV: `Per_Series` is set in salesline create path but not module create path.
  3. Mixed expression model in DEV: both new and legacy quantity formulas can exist by data region.
  4. Transformation sensitivity: detail DP pre-expands `Pieces`, so report-region assumptions must be consistent.
Recommended debugging workflow: trace one discrepant line by origin (salesline/module) -> check `McaCutlist` values -> inspect DP-written `McaCutlistTmp` values per report design -> evaluate exact SSRS textbox expression used.
